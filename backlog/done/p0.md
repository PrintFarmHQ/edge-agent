# PrintFarm Edge — Bambu Cloud Integration (Execution Tracker)

## Goal
Add production-grade Bambu Cloud support to edge-agent while preserving all existing non-Bambu flags and workflows.

## Locked Decisions (Current)
- Bambu support is enabled with `--bambu` (long-lived process, no standalone login command).
- Cloud is the source of truth for Bambu discovery/runtime.
- Edge-agent is the normalization layer for provider-specific states.
- SaaS adoption remains manual (no auto-adopt).
- Existing non-Bambu flags/functionality must remain unchanged.
- Local path for edge artifacts/credentials is `~/.printfarmhq/`.

## Current Phase Status
- [x] Phase 1: Cloud authentication (including MFA)
- [x] Phase 2: Cloud printer discovery + local/runtime integration
- [x] Phase 3: Print command execution for Bambu bindings
- [x] Phase 4: Remaining Bambu lifecycle commands

## Implemented (Shipped)

### Runtime and Auth
- [x] `--bambu` runtime flag support (with existing flags preserved).
- [x] Interactive login flow when token is missing/invalid.
- [x] Hidden password input in terminal mode (no echo).
- [x] Interactive MFA prompt when challenge is required.
- [x] Token-first startup flow:
  - [x] reuse valid access token
  - [x] refresh when expired and refresh token exists
  - [x] fallback to interactive login + MFA
- [x] Persist Bambu credentials to `~/.printfarmhq/bambu/credentials.json` with secure permissions.
- [x] Startup fails fast on unrecoverable Bambu auth failure.

### Discovery and Inventory
- [x] Bambu cloud device listing integrated in discovery loop.
- [x] Periodic inventory reporting (default 30s) includes Bambu entries.
- [x] Inventory submission includes `reachable`, `unreachable`, and `lost`.
- [x] Bambu discovery endpoint format normalized as `bambu://<device_id>`.
- [x] Bambu runtime binding snapshots resolved via cloud (no `BAMBU_CONNECT_URI` dependency).
- [x] Offline Bambu bindings are projected as connectivity-unreachable/offline semantics.
- [x] Offline Bambu snapshot condition no longer spams repeated hard error audits each poll.

### State Normalization
- [x] Canonical runtime states enforced for provider output:
  - printer: `idle | queued | printing | paused | error`
  - job: `pending | printing | completed | failed | canceled`
- [x] Bambu cloud status mapping implemented in edge-agent.
- [x] Moonraker/Klipper normalization path retained and compatible.

### SaaS Integration (Implemented alongside agent)
- [x] Discovery inventory accepts and stores Bambu native endpoints.
- [x] Bambu `unreachable` inventory entries are shown as pending adoption.
- [x] Bambu `unreachable` inventory entries are adoptable manually.
- [x] Adopting unreachable Bambu initializes printer as `offline`.
- [x] Adopting reachable entries initializes printer as `idle`.
- [x] `lost` entries remain non-adoptable.
